# Twitter Sentiment Analysis using Hadoop MapReduce

This project implements a Hadoop MapReduce pipeline to classify the sentiment of a dataset of tweets into positive, negative, and neutral categories using a dictionary-based approach.

## Project Structure

* `prepare.py`: A preprocessing script used to extract a subset of 1000 balanced tweets from the larger Sentiment140 dataset to create a more manageable file (`tweets.csv`).
* `mapper.py`: Reads the input text line-by-line, ignores the CSV header, tokenizes the text, and measures sentiment by counting predefined positive and negative keywords. Emits a key-value pair of the resulting sentiment (e.g., `positive 1`).
* `reducer.py`: Aggregates the emitted sentiment keys and sums up the total occurrences of each category perfectly.
* `tweets.csv`: A sample 1000-line dataset generated by the preparation script for quick testing and validation.
* `hadoop commands.txt`: Contains all necessary shell commands for initiating the Hadoop environment in Docker, interacting with HDFS, running local tests, and deploying the full Hadoop Streaming job.

## Execution

Local Testing (Standard I/O):
```bash
cat tweets.csv | python mapper.py | sort | python reducer.py
```

Hadoop Execution (using Hadoop Streaming):
```bash
hdfs dfs -rm -r /output
hadoop jar /opt/hadoop-3.2.1/share/hadoop/tools/lib/hadoop-streaming-3.2.1.jar \
-input /sentiment/tweets.csv -output /output \
-mapper "python3 /mapper.py" -reducer "python3 /reducer.py"
```

## Results

To view the output of the MapReduce job from HDFS:
```bash
hdfs dfs -cat /output/part-00000
```
